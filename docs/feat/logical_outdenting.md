Logseq의 "Logical Outdenting"은 Shift+Tab으로 블록을 outdent(들여쓰기 해제)할 때 아웃라인의 논리적 계층 구조를 고려하는 에디터 설정 옵션입니다. 이 기능은 기본적으로 Editor 설정에서 토글 가능하며, outliner 특성을 강화합니다.

## 옵션 꺼짐 (Direct Outdent)
기본 위치에서 바로 들여쓰기를 해제합니다. 
- 블록이 현재 위치에서 상위 레벨로 단순 이동
- 빠르고 직관적이나 계층 구조 변화가 적음

## 옵션 켜짐 (Logical Outdent)
아웃라인의 논리적 부모-자식 관계를 유지하며 outdent합니다.
- 자식 블록이 있으면 목록 맨 아래로 이동해 부모가 되도록 재배치
- 계층 구조 보존에 유리하나 일부 사용자에게는 불편(예: 예상치 못한 이동)

## 차이점 비교
| 상황 | 꺼짐 (Direct) | 켜짐 (Logical) |
|------|---------------|----------------|
| 자식 없는 블록 | 현재 위치 상위로 이동 | 현재 위치 상위로 이동 |
| 자식 있는 블록 | 현재 위치 상위로 이동 | 목록 끝으로 이동해 부모화 |
| 장점 | 빠르고 단순 | 논리적 계층 유지 |

설정은 Logseq의 Editor > Logical Outdenting에서 변경 가능하며, 실시간 협업 시 일관성을 위해 한 가지 동작만 권장됩니다.
